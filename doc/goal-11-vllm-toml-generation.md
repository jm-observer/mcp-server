# Goal 11: 辅助工具 - TOML 配置生成

## 目标

将采集到的 help 文本和 tool 配置 JSON Schema 发送给 vLLM，由 LLM 为每个子命令生成 tool 配置并判断安全性，最终输出合法的 `.toml` 文件。

## 前置依赖

- Goal 07（JSON Schema 输出）
- Goal 10（Help 递归解析、FlatCommand 列表、LlmClient）

## 涉及文件

```
mcp-tool-generator/src/
├── llm_client.rs         # 复用 Goal 10 的 vLLM 客户端
├── prompt.rs             # 新增 TOML 生成 prompt 模板
├── toml_output.rs        # TOML 文件生成与输出
└── main.rs               # 串联完整流程
```

## 实现方式

### Prompt 构建（prompt.rs）

为每个子命令构建 TOML 生成 prompt（与 Goal 10 的子命令识别 prompt 分开）：

```rust
pub fn build_toml_generation_prompt(
    command: &FlatCommand,
    json_schema: &str,
) -> Vec<ChatMessage>;
```

**System Prompt：**

```
你是一个 MCP tool 配置生成器。根据命令的 --help 输出，生成符合给定 JSON Schema 的 tool 配置。

规则：
1. 为命令生成一个 [[tools]] 配置块（TOML 格式）
2. 从 help 文本中提取参数，定义为 [[tools.parameters]]
3. 使用 ${var} 占位符引用参数
4. 判断命令安全性：
   - safe: 只读操作、查询、显示信息
   - dangerous: 删除、修改、写入、发送、部署等有副作用的操作
5. 对 dangerous 命令，在 TOML 前加 # DANGEROUS 注释

JSON Schema:
<schema>
{schema_content}
</schema>
```

**User Prompt：**

```
命令: {full_command}

Help 输出:
{help_text}

请生成该命令的 tool 配置（TOML 格式）。
```

### 调用策略

逐个调用：每个子命令单独一次 LLM 调用，简单可靠。

### LLM 输出解析

LLM 返回的文本中提取 TOML 块：
- 查找 ` ```toml ` 和 ` ``` ` 之间的内容
- 或查找以 `[[tools]]` 开头的内容
- 提取安全性标记（`# DANGEROUS` 或 LLM 在响应中的 safe/dangerous 判断）
- 对提取的 TOML 做基本合法性校验（尝试 `toml::from_str` 解析）

```rust
pub struct ToolOutput {
    pub toml_block: String,       // LLM 生成的 TOML 文本
    pub is_dangerous: bool,       // LLM 判断的安全性
    pub command: Vec<String>,     // 原始命令路径
}

/// 从 LLM 响应中解析 TOML 块和安全性标记
pub fn parse_llm_response(response: &str) -> Result<ToolOutput>;
```

### TOML 输出（toml_output.rs）

```rust
/// 将所有 tool 输出合并为最终的 .toml 文件
pub fn generate_toml_file(
    command_name: &str,
    outputs: &[ToolOutput],
) -> String;
```

输出格式：

```toml
# Auto-generated tool configuration for: cargo
# Generated by mcp-tool-generator

[config]
working_dir = "/data/repos"

# --- safe tools ---

[[tools]]
name = "cargo_build"
description = "Compile the current package"
command = "cargo"
args = ["build"]
# ...

# --- dangerous tools (uncomment to enable) ---

# # DANGEROUS: This command modifies files
# [[tools]]
# name = "cargo_clean"
# description = "Remove generated artifacts"
# command = "cargo"
# args = ["clean"]
```

- safe 的 tool 正常输出
- dangerous 的 tool 整块用 `#` 注释，用户手动取消注释启用

### 完整流程（main.rs）

```
1. 解析 CLI 参数
2. 启动 mcp-server（stdio），建立 MCP 连接
3. 获取 JSON Schema（mcp-server --schema 或库函数）
4. 递归爬取命令 help 文本（LLM 识别子命令）
5. 扁平化为 FlatCommand 列表
6. 逐个调用 vLLM 生成 tool 配置
7. 合并输出为 .toml 文件
8. 写入文件或输出到 stdout
9. 关闭 MCP 连接
```

### 依赖

复用 Goal 10 引入的依赖：
- `reqwest`：HTTP client（调用 vLLM API）
- `toml`：验证生成的 TOML 合法性
- `serde` + `serde_json`：序列化
