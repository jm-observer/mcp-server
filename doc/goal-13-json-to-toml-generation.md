# Goal 13: LLM 输出 JSON 反序列化生成 TOML 配置

## 目标

重构 TOML 配置生成流程：让 LLM 输出符合 `ToolDef` JSON Schema 的 JSON 文本，程序侧反序列化为结构体，再序列化为 TOML。替代当前 LLM 直接输出 TOML 文本 + 字符串截取的方式。

## 动机

- LLM 生成 JSON 的准确率远高于 TOML（训练数据中 JSON 占比更大）
- `serde_json::from_str` 反序列化提供结构化校验，字段缺失/类型错误直接报错，不再容忍格式错误
- `toml::to_string_pretty` 保证输出格式一致可控，不受 LLM 输出风格影响
- `ToolDef` 已有 `Serialize`、`Deserialize`、`JsonSchema` derive，基础设施就绪

## 前置依赖

- Goal 07（JSON Schema 输出）
- Goal 11（当前 TOML 生成流程，本次重构对象）

## 数据流变更

### 当前流程

```
help_text → prompt(要求输出 TOML) → LLM → 原始文本
  → parse_llm_response(字符串截取 TOML 块) → ToolOutput { toml_block: String }
  → generate_toml_file(手动拼接) → .toml 文件
```

### 目标流程

```
help_text → prompt(要求输出 JSON) → LLM → 原始文本
  → 提取 JSON 字符串 → serde_json::from_str::<ToolDef>() → ToolDef 结构体
  → 组装 ToolFile → toml::to_string_pretty(&tool_file) → .toml 文件
```

## 实现方式

### 1. `ToolDef` 增加 `dangerous` 字段（tool.rs）

在 `ToolDef` 上新增 `dangerous` 字段，用 `#[serde(skip_serializing)]` 标记，使其仅在 JSON 反序列化（从 LLM 输出解析）时生效，序列化到 TOML 时自动跳过：

```rust
#[derive(Debug, Serialize, Deserialize, Clone, JsonSchema)]
pub struct ToolDef {
    pub name: String,
    pub description: String,
    #[serde(flatten, default)]
    pub action: ToolAction,
    pub env: Option<HashMap<String, String>>,
    pub timeout_secs: Option<u64>,
    #[serde(default)]
    pub parameters: Option<Vec<ParameterDef>>,
    #[serde(default, skip_serializing)]
    pub dangerous: bool,
}
```

- `skip_serializing`：写 TOML 时不会出现 `dangerous = true` 字段
- `default`：反序列化时缺失则默认 `false`
- LLM 在 JSON 输出中通过 `"dangerous": true/false` 标注安全性

### 2. `ToolFileConfig` 加 `Default` derive（tool.rs）

```rust
#[derive(Debug, Serialize, Deserialize, Clone, JsonSchema, Default)]
pub struct ToolFileConfig { ... }
```

### 3. Prompt 重写（prompt.rs）

#### `build_toml_generation_prompt` → `build_json_generation_prompt`

改为要求 LLM 输出单个 `ToolDef` 的 JSON：

**System Prompt：**

```
你是一个 MCP tool 配置生成器。根据命令的 --help 输出，生成符合给定 JSON Schema 的 tool 配置。

规则：
1. 输出一个符合 JSON Schema 的 JSON 对象（单个 tool 定义）
2. 从 help 文本中提取参数，放入 parameters 数组
3. 在 args 中使用 ${param_name} 占位符引用参数
4. 判断命令安全性，设置 dangerous 字段：
   - false: 只读操作、查询、显示信息
   - true: 删除、修改、写入、发送、部署等有副作用的操作
5. 只输出 JSON，不要输出任何解释文字

JSON Schema:
<schema>
{schema_content}
</schema>
```

**User Prompt：**

```
命令: {full_command}

Help 输出:
{help_text}

请生成该命令的 tool 配置（JSON 格式）。
```

#### `parse_llm_response` → `parse_json_response`

重写解析逻辑，从 LLM 响应中提取 JSON 并反序列化为 `ToolDef`：

```rust
pub fn parse_json_response(response: &str) -> anyhow::Result<ToolDef> {
    let json_str = extract_json(response)?;
    let tool_def: ToolDef = serde_json::from_str(&json_str)?;
    Ok(tool_def)
}
```

JSON 提取策略（优先级从高到低）：
1. 提取 ` ```json ` 代码块内容
2. 尝试直接 `serde_json::from_str` 整段响应
3. 查找第一个 `{` 到最后一个 `}` 之间的内容

反序列化失败直接返回 `Err`，由调用方决定是否重试。

### 4. `ToolOutput` 简化（types.rs）

```rust
#[derive(Debug, Clone)]
pub struct ToolOutput {
    pub tool_def: ToolDef,
    pub command: Vec<String>,
}
```

`is_dangerous` 和 `toml_block` 字段移除：
- 安全性信息已在 `ToolDef.dangerous` 中
- TOML 文本由序列化生成，不再需要存储

### 5. TOML 输出简化（toml_output.rs）

```rust
pub fn generate_toml_file(command_name: &str, outputs: &[ToolOutput]) -> String {
    let mut safe_defs = Vec::new();
    let mut dangerous_defs = Vec::new();

    for o in outputs {
        if o.tool_def.dangerous {
            dangerous_defs.push(&o.tool_def);
        } else {
            safe_defs.push(&o.tool_def);
        }
    }

    // 生成 safe 部分：正常序列化
    let safe_file = ToolFile {
        config: Some(ToolFileConfig {
            working_dir: Some(".".into()),
            ..Default::default()
        }),
        tools: safe_defs.into_iter().cloned().collect(),
    };
    let mut out = format!(
        "# Auto-generated tool configuration for: {}\n\
         # Generated by mcp-tool-generator\n\n",
        command_name
    );
    out += &toml::to_string_pretty(&safe_file).unwrap();

    // 生成 dangerous 部分：序列化后每行加 # 注释
    if !dangerous_defs.is_empty() {
        out += "\n\n# --- dangerous tools (uncomment to enable) ---\n\n";
        for def in dangerous_defs {
            let single = ToolFile {
                config: None,
                tools: vec![def.clone()],
            };
            let toml_str = toml::to_string_pretty(&single).unwrap();
            // 跳过空 config，只取 [[tools]] 部分
            for line in toml_str.lines() {
                out += "# ";
                out += line;
                out += "\n";
            }
            out += "\n";
        }
    }

    out
}
```

### 6. `generate_tool_toml.rs`（bin）调整


## 注意事项

### flatten 兼容性

`ToolAction` 使用 `#[serde(flatten, tag = "type")]`，`toml` crate 对 flatten enum 的支持需要验证。如果 `toml::to_string_pretty` 序列化结果不符合预期，备选方案：
- 升级 `toml` crate 版本
- 手动处理 action 部分的序列化

### LLM 输出 JSON Schema 中 dangerous 字段的可见性

`dangerous` 字段使用 `skip_serializing` 而非 `skip`，因此 `JsonSchema` derive 仍会将其包含在 schema 输出中，LLM 能看到该字段定义并正确填写。
